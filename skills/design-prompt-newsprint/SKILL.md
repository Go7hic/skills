# Design Prompt Newsprint

## Overview

Use this skill when the user wants the Newsprint aesthetic. Keep the target product's content and requirements intact, then translate this style into the requested stack or artifact.

Stark black and white, high contrast, tight grids, newspaper aesthetic, sharp lines, editorial depth.


Read [references/style.md](references/style.md) before implementation whenever the task depends on this aesthetic.

## When to Use

- Requests for the Newsprint style or close synonyms tied to this aesthetic
- New landing pages, marketing sites, dashboards, or polished visual explorations
- Re-skinning an existing page while preserving the user's IA and content

## Working Rules

- Treat the style as direction, not as a literal Acme clone.
- Preserve accessibility, responsive behavior, and existing product constraints.
- If the task lives inside an existing design system, adapt this style without breaking local patterns.

## Prompt Starter

- Design a [surface] for [brand or product] in the "Newsprint" style.
- Keep the visual system light-first with #1a1a1a as the anchor accent.
- Let display typography carry the hierarchy.
- Start from these section archetypes when relevant: hero, stats, features, blog, how-it-works, benefits, testimonials, pricing, faq, footer.
- Translate the style to the user's domain and copy; do not recreate the Acme demo literally.
- Preserve any existing product constraints, content model, and accessibility requirements.
