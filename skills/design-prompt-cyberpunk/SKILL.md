# Design Prompt Cyberpunk

## Overview

Use this skill when the user wants the Cyberpunk aesthetic. Keep the target product's content and requirements intact, then translate this style into the requested stack or artifact.

High contrast neon on black, glitch animations, terminal/monospace fonts, tech-oriented decorations. A dystopian digital aesthetic inspired by 80s sci-fi and hacker culture.


Read [references/style.md](references/style.md) before implementation whenever the task depends on this aesthetic.

## When to Use

- Requests for the Cyberpunk style or close synonyms tied to this aesthetic
- New landing pages, marketing sites, dashboards, or polished visual explorations
- Re-skinning an existing page while preserving the user's IA and content

## Working Rules

- Treat the style as direction, not as a literal Acme clone.
- Preserve accessibility, responsive behavior, and existing product constraints.
- If the task lives inside an existing design system, adapt this style without breaking local patterns.

## Prompt Starter

- Design a [surface] for [brand or product] in the "Cyberpunk" style.
- Keep the visual system dark-first with #00ff88 as the anchor accent.
- Use a clean sans-led hierarchy with confident sizing.
- Start from these section archetypes when relevant: hero, stats, product-details, features, blog, how-it-works, benefits, testimonials, pricing, faq, footer.
- Translate the style to the user's domain and copy; do not recreate the Acme demo literally.
- Preserve any existing product constraints, content model, and accessibility requirements.
